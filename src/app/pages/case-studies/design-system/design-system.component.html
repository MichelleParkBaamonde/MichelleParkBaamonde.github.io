Building a design system for an automotive retailer

Timeline: January - July 2024

Role: Senior UX Designer

Team: Design Lead (2), Senior UX Designer (2)

Problem
The POS application was built in six separate workstreams. There was no design system in place during the team’s Sketch to Figma migration, so each Design Lead created their own components. Our team identified inconsistencies across different areas of the application.
Solution
Implement the new design system to help current and future designers design more efficiently and ensure UX/UI consistency across the client’s desktop and mobile applications.
Approach
Collaborated with designers on auditing existing files to create a common component library as a starting point, consolidated and removed redundant components, and created local components and supporting variants for our workstream.

Impact
  • Utilizing pre-made components and patterns improves productivity and reduces time and effort for designers when delivering ongoing enhancements to the POS application.
  • Providing clear categories and documentation in the design system allows new designers to onboard more smoothly and understand how the components are meant to be used.
  • Using one source of truth across the client’s design organization unifies the team and maintains UX/UI consistency.
